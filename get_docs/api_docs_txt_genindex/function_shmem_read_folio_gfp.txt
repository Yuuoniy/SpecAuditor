Function: struct folio * shmem_read_folio_gfp ( struct address_space * mapping , pgoff_t index , gfp_t gfp )

read into page cache, using specified page allocation flags.

Parameters
  struct address_space *mapping
    the folio’s address_space
  pgoff_t index
    the folio index
  gfp_t gfp
    the page allocator flags to use if allocating

Description
  This behaves as a tmpfs “read_cache_page_gfp(mapping, index, gfp)”,
with any new page allocations done using the specified allocation flags.
But read_cache_page_gfp() uses the -> read_folio() method: which does not
suit tmpfs, since it may have pages in swapcache, and needs to find those
for itself; although drivers/gpu/drm i915 and ttm rely upon this support.
